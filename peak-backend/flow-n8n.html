N8N + Docker OTP Flow (Easy Guide)

Purpose

Is document ka maqsad simple tareeqay se samjhana hai:

  1. Hum Docker aur n8n kyun use kar rahe hain,
  2. Setup kaise karna hai,
  3. Client ki email se OTP kaise read karke backend ko bhejna hai.

1) Hum Docker kyun use kar rahe hain?

  • Windows pe kuch Node packages (native build) install issues dete hain.
  • Docker se ready-made container chalta hai aur local dependency errors avoid hotay hain.
  • Environment stable rehta hai: team ke sab members same setup use kar sakte hain.
  • Easy start/stop: docker compose up -d / docker compose down

2) n8n kya role play karta hai?

n8n humara automation engine hai:

  • Jab OTP email aaye - trigger chale,
  • Email content se 6-digit OTP nikaale,
  • OTP humare backend webhook ko POST kare.

Simple logic: "Jab X ho, to Y karo"

  • X = OTP email aayi
  • Y = OTP extract karke backend ko bhejo

3) High-Level End-to-End Flow

  1. Client side se carrier OTP emails automation@peakinsurance.com par forward hoti hain.
  2. n8n Email Trigger inbox monitor karta hai.
  3. n8n Code node email text me se OTP regex se nikalta hai.
  4. n8n HTTP Request node backend endpoint hit karta hai: POST /webhooks/otp
  5. Backend OTP receive karta hai aur automation flow ko continue karta hai.

4) Docker + n8n Setup Steps

  1. Docker Desktop install + engine running confirm.
  2. C:\n8n-docker folder me docker-compose.yml create karo.
  3. Compose run karo: docker compose up -d
  4. Browser me open karo: http://localhost:5678
  5. n8n owner account create karo.

5) n8n Workflow Steps (Practical)

Workflow name: Travelers OTP -> Backend Webhook

Node 1: Email Trigger

  • Mailbox: INBOX
  • Credential: IMAP/Outlook/Gmail (jo provider ho)
  • Poll interval: 1 min
  • Optional filter: subject contains OTP/verification

Node 2: Code (OTP Extract)

  • Email subject/body read karo
  • Regex use karo: \b(\d{6})\b
  • Output: carrier, otp, extractedAt, valid

Node 3: IF

  • valid == true ho to next step
  • false ho to ignore ya alert

Node 4: HTTP Request (Backend Webhook)

  • Method: POST
  • URL: http://host.docker.internal:7001/api/quotes/webhooks/otp
  • Header: x-webhook-secret: <WEBHOOK_SECRET>
  • JSON body: carrier, otp, extractedAt

Node 5: Activate Workflow

  • Workflow ON rakho taake real-time emails process hon.

6) Client se kya chahiye?

  • automation@peakinsurance.com mailbox access (password/app-password)
  • Mail provider details (Outlook/Gmail/custom IMAP)
  • Agar custom IMAP ho: host, port, SSL details
  • OTP sender examples (2-3 sample emails/senders/subjects)

7) Client ko forwarding me kya karna hai?

Case A: OTP email par aata hai

  • Outlook/Gmail rule set karein
  • Condition: sender = carrier OTP sender
  • Action: forward to automation@peakinsurance.com

Case B: OTP SMS par aata hai

  • Phone me SMS-to-email auto-forward setup karein
  • Ya Twilio fallback use karein

8) Security Rules (Must)

  • Credentials chat me share na karein.
  • Real secrets code ya git me commit na karein.
  • WEBHOOK_SECRET must use karein.
  • Logs me raw OTP/password show na karein.

9) Verification Checklist

  • [ ] Docker engine running
  • [ ] n8n UI open on localhost:5678
  • [ ] Email Trigger test successful
  • [ ] OTP extract test successful (6-digit)
  • [ ] Backend webhook returns 200
  • [ ] Workflow activated

Short Conclusion

Humne Docker isliye use kiya taake n8n stable aur quickly run ho. n8n inbox monitor karega, OTP extract karega, aur backend ko send karega. Is se OTP automation manual steps ke baghair chal sakti hai, jab client forwarding properly set kare.
