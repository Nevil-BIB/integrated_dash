Peak Quote - Complete UI & Data Flow Guide

Yeh document new developer onboarding ke liye hai. Ismein upload se le kar extraction, review/edit/save, quote preview, APIs, aur home / auto / both ka complete flow clearly explain kiya gaya hai.

Scope: Current implementation based on project APIs/components.

1) High-Level End-to-End Flow

User selects PDF + insurance type (home/auto/both) -> POST /api/upload (file storage + extraction row create) -> POST /api/extract (download PDF, convert pages, AI extraction, save extracted_data) -> Redirect /review/[id] -> Render extraction form(s) (home/auto/both) -> User edits fields -> Auto-save PATCH /api/extractions/[id] -> Proceed to quote page /review/[id]/quote?type=... -> Quote validation + final preview -> Submit quote POST /api/quotes/submit

2) Step-by-Step: PDF Upload se kya hota hai

2.1 Frontend Upload UI

  • User file select/drop karta hai in PdfUploadZone.
  • User insurance type choose karta hai: home, auto, both.
  • useUpload hook progress/status handle karta hai.

2.2 Upload API

  • POST /api/upload
  • Auth check via Supabase session.
  • PDF type + max 20MB validation.
  • File Supabase Storage bucket fact-finders mein save hoti hai.
  • DB table extractions mein row banti hai with status = pending.

2.3 Extraction API

  • POST /api/extract with extractionId + insuranceType.
  • Extraction row processing mein set hoti hai.
  • Storage se PDF download hoti hai.
  • PDF pages images mein convert hoti hain (convertPDFToImages).
  • AI extraction run hota hai (details below).
  • Result extractions.extracted_data mein save hota hai, status completed.

3) Fields kahan se aati hain? (Source of Truth)

A) UI Field Definitions

  • Home fields: src/types/home-extraction.ts
  • Auto fields: src/types/auto-extraction.ts
  • Yahan labels, input types, required flags, options define hote hain.

B) AI Extraction Contract

  • Prompt: src/lib/openrouter/prompts.ts
  • Schema validation: src/lib/openrouter/schemas.ts
  • Yahan decide hota hai AI se kaunsi JSON structure expect karni hai.

C) API Data Types

  • src/types/extraction.ts (HomeApi/AutoApi/Combined).
  • Yeh backend storage/extraction shape define karta hai.

D) API -> UI Mapping

  • src/lib/extraction/transform.ts
  • DB/API format ko review/quote UI format mein convert karta hai.

Agar field UI mein add ho gayi lekin prompt/schema/transform mein add na ho, to auto-fill incomplete rahega.

4) Home vs Auto vs Both ka exact behavior

4.1 Home

  • Extraction API extractHomeFromImages call karta hai.
  • Saved data shape: Home API structure.
  • Review page par Home form load hota hai.

4.2 Auto

  • Extraction API extractAutoFromImages call karta hai.
  • Saved data shape: Auto API structure.
  • Review page par Auto form load hota hai.

4.3 Both

  • Home + Auto extraction parallel run hota hai.
  • Combined payload save hota hai with: shared, autoPersonal, home, auto, quoteType=both.
  • Review page tabs/sections ke through dono data edit hota hai.

5) Review Page: Data kaise show/edit/save hota hai

5.1 Data load

  • /review/[id] page extraction record fetch karta hai.
  • Client side form components normalized data render karte hain.

5.2 Edit behavior

  • User field edit karta hai (text/select/date etc.).
  • Conditional fields parent values ke basis par show/hide hoti hain.

5.3 Save behavior

  • Auto-save/debounced updates trigger hoti hain.
  • PATCH /api/extractions/[id] mein latest extracted_data save hota hai.
  • Yahi reason hai next visit par updated values milti hain.

5.4 Next time data kaise milta hai

  • Jab page reopen hota hai, server DB se extracted_data re-fetch karta hai.
  • Client usse form state mein hydrate karta hai.

6) Quote Preview aur Final Submit

6.1 Quote page load

  • /review/[id]/quote?type=home|auto|both
  • Raw DB data detect + transform hota hai before preview.

6.2 Validation

  • Field-level validation summary banayi jati hai.
  • Missing required fields highlight hote hain.

6.3 Submit

  • POST /api/quotes/submit
  • Validated fields ko structured quote payload mein convert karta hai.
  • quotes table mein quote record create hota hai.
  • Optional webhook payload generate/send hota hai.

7) APIs Used (Internal + External)

Internal APIs (Next.js)

  • POST /api/upload - PDF upload + extraction row create
  • POST /api/extract - PDF process + AI extraction + DB save
  • GET /api/extract?id=... - extraction status
  • GET /api/extractions/[id] - extraction fetch
  • PATCH /api/extractions/[id] - edited data save (auto-save)
  • DELETE /api/extractions/[id] - extraction + file delete
  • POST /api/quotes/validate - quote data validation
  • POST /api/quotes/submit - final quote submission

External Services / APIs

  • Supabase Auth - user session/auth checks
  • Supabase Storage - PDF file storage (fact-finders)
  • Supabase Database - extractions & quotes tables
  • OpenRouter Chat Completions API - Claude Vision extraction
  • Optional external webhook via WEBHOOK_URL for downstream automation
  • Optional Node backend integration via NEXT_PUBLIC_NODE_BACKEND_URL (quote generation/job status)

8) Data Storage Summary

  • PDF file: Supabase Storage bucket fact-finders.
  • Extracted JSON: extractions.extracted_data (JSONB).
  • Extraction lifecycle: pending -> processing -> completed/failed -> quoted.
  • Final quote payload: quotes.quote_data.

9) Common Debug Checklist (for New Developers)

  1. Field UI mein hai? Check home-extraction.ts/auto-extraction.ts.
  2. Prompt mein field demand ki? Check prompts.ts.
  3. Schema allow karta hai? Check schemas.ts.
  4. Default object mein field initialized hai? Check openrouter/client.ts and openrouter/defaults.ts.
  5. Transform map hua? Check transform.ts.
  6. Review page save ho raha? Check PATCH /api/extractions/[id].
  7. Quote page par stale data to nahi? Force fresh fetch + save-before-navigation confirm karein.

10) Final Concept (Simple Words)

Project ka core idea yeh hai: PDF ko pehle safe upload karna, phir AI se structured JSON nikalna, phir user ko editable form dena, phir edits ko DB mein persist karna, aur finally validated quote submit karna.

Is chain mein agar kisi step par field mapping miss ho jaye (prompt/schema/transform/UI), to auto-fill partial hota hai. Isi liye har new field ko end-to-end sync karna zaroori hai.
